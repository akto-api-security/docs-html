API reference

You will need the following inputs for each of the API endpoints:
  • YOUR_API_KEY : Go to settings/ integrations and fetch API key under external API.
  • URL : This is the url of your Akto dashboard.
  1. 1.
    api/getAPICollection: This endpoint is to fetch all the APIs of one collection
    • Query parameters: Collection_ID You can get this by going to a collection and copying the number from url.
    • Command: **** curl -H "X-API-KEY:`` YOUR_API_KEY" "URL/api/getAPICollection?apiCollectionId=Collection_ID"
    • Output: You will get list of all the params from all the urls in that collection under data.endpoints.
{
"data": {
"name": "Main application",
"endpoints": [
{
"apiCollectionId": 1641506725,
"isHeader": false,
"method": "POST",
"param": "category#id",
"responseCode": 200,
"subType": "INTEGER_32",
"timestamp": 1641509240,
"url": "https:\/\/petstore.swagger.io\/v2\/pet"
},
....
]
}
}
2. api/loadRecentParameters: This endpoint is to fetch all the new params of your complete API inventory.
  • Command: **** curl -H "X-API-KEY:`` YOUR_API_KEY" "URL/api/loadRecentParameters"
  • Output: You will get list of all the new params under data.endpoints.
{
"data": {
"name": "Main application",
"endpoints": [
{
"apiCollectionId": 1641506725,
"isHeader": false,
"method": "POST",
"param": "category#id",
"responseCode": 200,
"subType": "INTEGER_32",
"timestamp": 1641509240,
"url": "https:\/\/petstore.swagger.io\/v2\/pet"
},
....
]
}
}
3. api/loadSensitiveParameters: This endpoint is to fetch all the sensitive params of your complete API inventory.
  • Command: **** curl -H "X-API-KEY:`` YOUR_API_KEY" "URL/api/loadSensitiveParameters"
  • Output: You will get list of all the new params under data.endpoints.
{
"data": {
"name": "Main application",
"endpoints": [
{

"timestamp": 1641509240,
"url": "https:\/\/petstore.swagger.io\/v2\/pet"
},
....
]
}
}
4. api/fetchEndpointTrafficData: This endpoint is to fetch all the traffic data of a particular endpoint.
  • Query parameters:
    1. 1.
      Collection_ID You can get this by going to a collection and copying the number from url.
    2. 2.
      Endpoint_Url This is endpoint url for which you want to get the traffic. ( example: api/board")
    3. 3.
      Method The http method for which you want to get the traffic.
    4. 4.
      Start_Epoch The starting time stamp of the traffic you want to fetch. ( example: 1641748269)
    5. 5.
      End_Epoch The starting time stamp of the traffic you want to fetch. ( example: 1646932269)
  • Command: **** curl -H "X-API-KEY:YOUR_API_KEY``" -d '{"method":``Method, "url":`` Endpoint_Url, "apiCollectionId":`` Collection_ID, "startEpoch":Start_Epoch, "endEpoch":`` End_Epoch}' -H "Content-type: application/json" -X POST "URL/api/fetchEndpointTrafficData"
  • Output: You will get daily traffic trend under traffic key.
{
"traffic": {
"20220212": 19923,
"20220213": 20427,
"20220214": 21142,
"20220215": 20477,
...
}
}
5. api/fetchApiInfoList: This endpoint is to fetch metadata of all the APIs of one collection
  • Query parameters: Collection_ID You can get this by going to a collection and copying the number from url.
  • Command: **** curl -H "X-API-KEY:`` YOUR_API_KEY" "URL/api/fetchApiInfoList?apiCollectionId=Collection_ID"
  • Output: You will get list of all the params from all the urls in that collection under apiInfoList
{
"apiInfoList": [
{
"actualAuthType": ["BASIC"],
"allAuthTypesFound": [
["BASIC"]
],
"apiAccessTypes": ["PRIVATE"],
"id": {
"apiCollectionId": 0,
"method": "GET",
"url": "\/api\/getCardInsights"
},
"lastSeen": 1646763449,
"violations": {}
},
.....
]
}
6. api/fetchSampleData: This endpoint is to fetch all the sample data entries stored for a particular endpoint. Note that Akto stores only latest 10 samples for any endpoint.
  • Query parameters:
    1. 1.
      Collection_ID You can get this by going to a collection and copying the number from url.
    2. 2.
      Endpoint_Url This is endpoint url for which you want to get the traffic. ( example: api/board")
    3. 3.
      Method The http method for which you want to get the traffic.
  • Command: **** curl -H "X-API-KEY:YOUR_API_KEY``" -d '{"method":``Method, "url":`` Endpoint_Url, "apiCollectionId":`` Collection_ID}' -H "Content-type: application/json" -X POST "URL/api/fetchSampleData"
  • Output: You will get sample request response stored for the queried endpoint
{
"sampleDataList": {
{
"id": {
"apiCollectionId": "<apiCollectionId>",
"bucketEndEpoch": 0,
"bucketStartEpoch": 0,
"method": "<method>",
"responseCode": -1,
"url": "<endpointUrl>"
},
"samples": []
}
.....
}
}
